# Wake Word Setup Guide

This guide walks you through the exact process of adding custom wake words (for example, “Anna” and “Henry”) to the HomeBrain platform, delivering the trained Porcupine models to the hub, and activating them on remote devices.

---

## 1. Prerequisites

Before you begin, make sure you have:

- A **Picovoice Console** account (https://console.picovoice.ai/) with access to Porcupine wake-word training.
- The **hub (Jetson)** running the HomeBrain server.
- At least one **remote device** (e.g., Raspberry Pi) configured with the HomeBrain remote client.
- SSH access to both the hub and each remote device.
- The optional Porcupine binding installed on the remote device (`npm install @picovoice/porcupine-node`) and the Picovoice AccessKey exported (`PICOVOICE_ACCESS_KEY` or `PV_ACCESS_KEY`).

---

## 2. Train and Download Wake Word Models from Picovoice

Repeat this section for each wake word you want to deploy (e.g., “Anna”, “Henry”):

1. Sign in to https://console.picovoice.ai/.
2. Click **Create New Wake Word**.
3. Enter the wake-word phrase exactly as you want it recognized (e.g., `Anna`).
4. Choose **Porcupine (Wake Word)** as the engine.
5. Select **Platform → Raspberry Pi** (that’s the remote device target).
6. Click **Train** and wait for the status to finish (should display “Done”).
7. Click **Download**, then pick the `.ppn` file for the Raspberry Pi build. The filename usually looks like:
   ```
   Anna_en_raspberry-pi_v3_0_0.ppn
   ```
8. Save the `.ppn` file locally on your workstation (for example, `C:\Users\<you>\Downloads\anna_raspberry-pi.ppn` on Windows).

> **Tip:** Train and download a separate `.ppn` for every unique wake word. If you need multiple platform variants (desktop, Android, etc.), download each appropriate build.

---

## 3. Copy the `.ppn` Files onto the Hub

The HomeBrain hub serves wake-word models from `server/public/wake-words/`. Do the following on your workstation (PowerShell example shown):

1. Ensure the directory exists on the hub:
   ```powershell
   ssh matt@<hub-ip> "mkdir -p ~/homebrain/HomeBrainv2/server/public/wake-words/"
   ```
2. Copy each `.ppn` file to that directory:
   ```powershell
   scp "C:\Users\<you>\Downloads\anna_raspberry-pi.ppn" `
       matt@<hub-ip>:/home/matt/homebrain/HomeBrainv2/server/public/wake-words/
   ```
3. (Optional) Rename files to a clean slug if needed, e.g. `anna_raspberry-pi.ppn`, `henry_raspberry-pi.ppn`.
4. Verify the files arrived:
   ```powershell
   ssh matt@<hub-ip> "ls -lh ~/homebrain/HomeBrainv2/server/public/wake-words/"
   ```

> **Note:** These files are binary bundles generated by Picovoice. They must be downloaded from Picovoice—HomeBrain does not (and legally cannot) generate them on its own.

---

## 4. Restart Remote Devices to Pull Models

Each remote device downloads the wake-word models the next time it authenticates. Restart the service on **every** remote device:

```bash
ssh pi@<remote-device-ip>
sudo systemctl restart homebrain-remote
sudo journalctl -u homebrain-remote -f
```

Watch for log lines like:

```
Downloading wake word model for "Anna"...
Wake word detection active (Porcupine)
```

If you do not see the download message, confirm the `.ppn` file exists on the hub and that the remote device has the optional Porcupine binding installed plus a valid `PICOVOICE_ACCESS_KEY`.

---

## 5. Verify Detection

After the restart:

1. Speak the wake word (“Anna”) near the remote device.
2. Watch the logs (`journalctl -u homebrain-remote -f`). You should see:
   ```
   Wake word detected: "Anna" (confidence: 0.65)
   ```
3. Check the HomeBrain hub logs or UI for the corresponding wake-word event.

If the wake word does not trigger:

- Ensure the correct `.ppn` file matches the platform (Raspberry Pi).
- Confirm the remote device restarted successfully and has network access to the hub.
- Confirm the AccessKey is exported and valid (`echo $PICOVOICE_ACCESS_KEY`).
- Re-run `npm install @picovoice/porcupine-node` on the remote device if the module is missing.

---

## 6. Managing Multiple Wake Words

- Add as many wake words as you need by repeating the training and copying process (e.g., `henry_raspberry-pi.ppn`).
- The hub automatically matches a profile’s configured wake words to the available `.ppn` files and pushes them to each remote device.
- When you update or replace a `.ppn`, just overwrite the file on the hub and restart the remote device service—it will re-download the latest version.

---

## 7. Security & Maintenance Considerations

- Treat `.ppn` files as sensitive artifacts—they are licensed per Picovoice’s terms and tied to your account.
- Do not commit `.ppn` files to version control. Keep them in the `server/public/wake-words/` directory on the hub or in a secure artifact store.
- Regularly back up your `.ppn` files so you can restore them if the hub is rebuilt.
- Monitor Picovoice’s usage limits and billing terms if you generate many wake words.

---

## Summary Checklist

1. Train wake words in Picovoice Console and download the Raspberry Pi `.ppn`.
2. Copy the `.ppn` into `server/public/wake-words/` on the hub.
3. Restart each remote device (`sudo systemctl restart homebrain-remote`).
4. Confirm logs show the model download and wake-word detection.
5. Repeat for any additional wake words.

Once these steps are complete, your remote devices will respond to your custom wake words. If you plan to automate `.ppn` creation from user profiles in the future, you’ll need to integrate formally with Picovoice’s training APIs and license terms.
